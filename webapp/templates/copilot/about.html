How Luau Copilot Works

A three-stage self-improvement pipeline that trains, evaluates, and corrects a Roblox code generation model.

The Training Pipeline

1

SFT

Fine-tune on curated Luau + Claude gold data

2

RFT + Claude Judge

Score with 4 scorers + Claude Sonnet, keep the best

3

Self-Correction

Diagnose bugs, rewrite code, create new training data

Stage 1: Supervised Fine-Tuning

Fine-tune Devstral (24B) with QLoRA on a combined dataset: curated Luau scripts from the-luau-stack (reverse-labeled with Claude) plus gold training data generated by Claude Sonnet 4.5. The model learns Luau syntax, Roblox API patterns, and game development idioms.

Base Model

Devstral-Small-2505 (24B)

Training Data

the-luau-stack + Claude Sonnet 4.5

Stage 2: Reinforcement Fine-Tuning with Claude Sonnet

Generate multiple candidates per task, score each with four deterministic scorers (40%) plus Claude Sonnet as an LLM judge (60%). Keep the highest-scoring outputs and train on those. Cross-model judging prevents reward hacking.

Stage 3: Self-Correction Loop

The model generates code, four regex-based scorers diagnose issues, then the model rewrites its own code to fix them. Runs up to 3 rounds per task. Corrected outputs become new training data.

generate(task) → diagnose(code) → heal(code, issues) → re-score → repeat until passing
